## Phase Goal

**As a** first-time visitor opening the portfolio in a fresh tab, **I want to** be greeted by a single zen brush-stroke that uncovers the OS-like hero (and to be able to skip it instantly), **so that** the experience feels like booting into a crafted operating system rather than loading a web page.

> Derived from the ROADMAP Phase 2 goal + success criteria 1 and 2. The boot is the entry gesture; this slice makes the hero (already built in 02-01) appear *through* a reveal.

## MVP Slice Tasks

After Task 1 the brush stroke draws + wipes to reveal the hero, gated to play once per session and bypassed under reduced motion. After Task 2 the boot is skippable by any input and mounted app-wide without replaying on navigation. Each task advances the live boot behavior a visitor experiences.

## Trust Boundaries

| Threat ID | Category | Component | Disposition | Mitigation Plan |
