## Color

**INHERITED in full** from Phase 1 — dichromatic minimalism (near-black `#020202` + ice-blue `#B2D5E5`),
both themes, all glass/gradient/glow tokens. No third hue. Phase 2 adds NO new color tokens. Reproduced
summary:

| Role | Dark value | Light value | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | `#020202` | `#F4F7F9` | Page/boot void, hero background base (`bg-background`) |
| Secondary (30%) | `#0B0D0F` + glass fills | `#FFFFFF` + glass fills | (Not heavily used in this phase — no cards/dock) |
| Accent (10%) | `#B2D5E5` | `#3E7C93` | See reserved-for list below (`var(--primary)`) |
| Destructive | `#E5746A` | `#C0463B` | None in this phase (no forms/destructive actions) |

**Accent reserved-for in Phase 2 (a strict subset of the inherited 10% reserved list — never "everything"):**
- The boot brush-stroke ink tint — accent may **subtly** tint the stroke over the `#020202` void (D-02), staying within the two-hue palette. Default treatment: the stroke renders in near-white/foreground ink with a soft accent-tinted glow edge (`--glow-accent`), reading as a single ice-tinted gesture, not a saturated blue line. Saturated accent stays well under 10% of the frame.
- The hero ambient lighting hue: aurora gradient (`--gradient-aurora`, already accent-tinted) and the blurred floating-light blobs use the accent hue at the low opacities baked into the token (0.18 dark / 0.12 light) — this is the "glow/aurora lighting hue" already on the inherited reserved list.
- The particle dots: rendered in foreground/accent at very low opacity (≤ ~0.4), part of the ambient lighting — not saturated fills.
- A subtle accent glow on the wordmark (`[text-shadow:var(--glow-accent)]`, the pattern already used on the Phase 1 placeholder `h1`) — key brand emphasis, on the inherited "brand marks / hover-glow" reserved list.
- The scroll-cue glyph may use the accent hue (key affordance) or `text-muted-foreground`; default `text-muted-foreground` with an accent glow on hover, to keep accent under budget.

**Accent is NOT used for** (INHERITED): body/tagline text (uses `text-muted-foreground`), title text
(`text-foreground`), large flat fills, or the boot/hero background base (always `bg-background` = the void).
The hero must remain **subtle** — no neon/cyberpunk saturation (D-07, CLAUDE.md). Saturated accent stays
under ~10% of any viewport.

## Boot Animation (NEW — D-01..D-05, D-12)

The boot is a **single calligraphic brush stroke** that sweeps across the `#020202` void and "wipes" away
via an SVG stroke mask to reveal the hero beneath it (D-01). One zen gesture — it must feel like the stroke
*uncovering* the OS, not a separate loading screen.

### Visual / structure
- **Overlay:** a `fixed inset-0 z-50` full-bleed client component (`'use client'`) over `bg-background`
  (`#020202`, the void — D-02). The hero renders underneath from first paint (SSR-friendly); the boot overlay
  must NOT block first paint of the underlying hero, so skip/reduced-motion can reveal it instantly.
- **Stroke:** one `<svg>` `<path>` of a calligraphic brush sweep. The visible "ink" is foreground/near-white
  with a soft accent-tinted glow edge (`var(--glow-accent)`) — reads as a single ice-tinted gesture within
  the two-hue palette (D-02). No third hue, no saturated blue line.
- **Reveal mechanism:** the stroke is used as a **mask/wipe** — as the stroke completes its sweep it wipes the
  opaque void away to expose the hero (the gesture uncovers the OS). Implement via SVG `mask` +
  `stroke-dasharray`/`stroke-dashoffset` animation (a transform/opacity-safe technique) or an animated
  clip/mask reveal — see Motion/perf rules below.

### Motion spec (extends the inherited motion contract — D-12)
| Aspect | Spec |
|--------|------|
| Engine | Framer Motion (locked engine), SVG path. |
| Properties animated | `transform` / `opacity` and SVG `pathLength`/`stroke-dashoffset` (Framer Motion's `pathLength` is GPU-friendly and the sanctioned way to animate an SVG stroke draw). **No** animating width/height/top/left/box-shadow in the loop (FND-06, D-07). |
| Easing | `[0.22, 1, 0.36, 1]` family (INHERITED easeOutExpo-like) — D-12. |
| Stroke sweep duration | **~2.0s** (DEFAULT — within the D "target ~2–2.5s" window; chosen at the faster end so recruiters aren't kept waiting). The wipe/reveal of the hero overlaps the tail of the sweep so total boot-to-hero is ≤ ~2.4s. |
| Reveal/wipe | After the stroke completes, the opaque void fades/wipes out over `~0.4s` opacity, easing the same family, handing off to the hero entrance choreography. |
| `will-change` | Used sparingly on the animating path only; removed after boot completes. |

### Hero entrance choreography (NEW — D-12, discretion)
After the boot wipe (or immediately, on skip/reduced-motion), the hero elements enter using the inherited
`Reveal` contract (fade + 16px upward translate, 0.5s, ease `[0.22,1,0.36,1]`):
- Stagger order: wordmark → title → tagline → scroll cue, staggered **0.08s** (DEFAULT — slightly longer than
  the inherited 0.06s group stagger for a more deliberate hero entrance; still within the subtle envelope).
- Reuse `RevealGroup` / `RevealItem` where it fits (D-12); the background ambient layers start their loops
  immediately (they are behind the wipe).

### Skip & session replay (D-03, D-04 — LOCKED)
| Behavior | Spec |
|----------|------|
| Skip triggers | Any `click`, `keydown`, or `scroll`/`wheel`/`touchmove` immediately skips → jumps straight to the hero (D-04). Listeners attach on mount, detach on completion/skip. |
| Auto-advance | If not skipped, boot auto-advances to the hero after its duration (D-04). |
| Replay scope | Plays **once per browser tab session**, tracked via `sessionStorage` (e.g. key `careeros:boot-played`) — D-03. On boot completion or skip, set the flag. |
| No replay on route change | Does NOT replay on client-side (App Router) navigations within the same tab session (D-03 / BOOT-02 "no disruptive replay"). |
| Fresh replay | Replays on a fresh visit / new tab (new session) — D-03. |

### Reduced motion (D-05 — LOCKED)
- Under `prefers-reduced-motion: reduce`, the boot **does NOT animate** — the hero is shown immediately, no
  stroke sweep (D-05). Gate via the same `useReducedMotion()` pattern as `src/components/motion/reveal.jsx`
  (D-12). In this case the overlay either never mounts or unmounts instantly (≤0.01s opacity), and the
  `sessionStorage` flag may be set so the (already-skipped) boot doesn't re-evaluate disruptively.

## Hero Background (NEW — D-06, D-07)

Built with **CSS/SVG + Framer Motion only — no canvas, no new dependencies** (D-06). Three stacked ambient
layers behind the hero content, all animating **`transform`/`opacity` only** (GPU-friendly, D-07, FND-06),
all **subtle** (no neon/cyberpunk, D-07), all degrading to **static** under reduced motion (D-07).

| Layer (back → front) | Technique | Spec |
|----------------------|-----------|------|
| (a) Aurora | Animated radial gradient using the inherited `--gradient-aurora` token (`bg-gradient-aurora`) | One or two stacked gradient layers slowly drifting via `transform` (translate/scale) and gentle `opacity` breathing. Loop **~18s** (DEFAULT — within the inherited "ambient/aurora loops 8–20s" band, at the slow end for calm). Accent hue is already baked into the token at low opacity. |
| (b) Floating light | A few large **blurred divs** (`blur-3xl`/heavy `filter: blur`) tinted with the accent hue at low opacity (token-derived) | **3 blobs** (DEFAULT — "a few", keeps DOM/paint light for 90+ Lighthouse headroom). Each drifts on a slow independent `transform` translate loop, **~14–20s**, slightly different offsets/phases. Positioned off-center so light pools behind the wordmark. |
| (c) Particles | A **light CSS/SVG dot layer** (~20–40 dots — D-06) | **~28 dots** (DEFAULT — mid of the 20–40 band; tuned subtle + Lighthouse-safe). Small (1–3px) foreground/accent dots at low opacity (≤ ~0.4), each on a slow `transform` (translateY/translateX drift) + gentle `opacity` twinkle loop, **~10–20s**, staggered phases. No mouse-follow/parallax (that's MOT-02, Phase 6). Prefer a single SVG or a small mapped array of absolutely-positioned divs. |

### GPU / performance rules (D-07, FND-06 — INHERITED contract)
- Animate **only** `transform` + `opacity`. Never animate `width`/`height`/`top`/`left`/`box-shadow`/`filter` values inside the loops (set blur once, animate the blurred element's transform).
- `will-change: transform` used sparingly on the floating-light blobs and aurora layer; avoid on all 28 particles at once.
- Keep total animated node count modest (3 blobs + ~28 dots + 1–2 aurora layers) to protect the 90+ Lighthouse target (formal audit is Phase 7, but build GPU-friendly from the start).
- The whole background sits behind hero content (`-z-10`/absolute) and `pointer-events-none` so it never intercepts the boot-skip or scroll-cue interactions.

### Reduced motion (D-05/D-07 — INHERITED gate)
- Under `prefers-reduced-motion: reduce`, all three layers render **static**: aurora and blobs hold a fixed
  position (no drift), particles hold fixed opacity/position (no twinkle/drift). Gate via `useReducedMotion()`
  (the inherited pattern). The static composition must still look intentional and premium (a calm aurora-lit
  void), not broken.

## Scroll Cue (NEW — D-10)

A subtle animated scroll-down cue at the bottom of the hero — the entry affordance (D-10). NOT a ⌘K hint
(Phase 6) and NOT a left-aligned layout (D-10).

| Aspect | Spec |
|--------|------|

| Position | `absolute bottom-12` (2xl/48px from bottom), horizontally centered in the hero. |
| Color | `text-muted-foreground` at rest; accent glow (`drop-shadow`/`--glow-accent`) on hover/focus (keeps accent under budget). |
| Motion | Gentle continuous **vertical bob** — `y` translate loop (e.g. `0 → 8px → 0`) and/or soft opacity pulse, **~1.8s** loop, ease in-out. `transform`/`opacity` only (FND-06). |
