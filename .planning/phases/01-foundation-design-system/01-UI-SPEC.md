# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **FOUNDATION PHASE.** This is the canonical design system for the entire "Career OS" portfolio. Every later phase (boot/hero, dashboard/dock, about/skills, projects/experience, transmission center/command palette) inherits these tokens, type scale, color contract, motion baseline, and copy voice. Phase 1 ships the tokens + theme system + motion wrapper that prove this contract on a placeholder section — it does NOT build the named sections (those are Phases 2–6). When a later phase needs a token not declared here, it extends this file; it must not redefine these primitives.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (ShadCN UI) |
| Preset | `standard-next-tailwind-dark` — Next.js App Router + Tailwind + class-based dark mode. NOT YET INITIALIZED. Running `npx shadcn init` against the scaffolded app (CSS variables: yes, base color: neutral, dark mode: class) is an execution task for the planner. components.json does not exist yet — Phase 1 creates it. |
| Component library | radix (shadcn's default primitive layer) |

Mono is never used for general body copy. Both loaded via `next/font` (self-hosted, `display: swap`, CSS-variable bound: `--font-geist-sans`, `--font-geist-mono`). |

Deployment: Vercel. Avoid `@vercel/og` dynamic OG/icon routes (Node-box crash) — use static OG/icon assets.

**Token delivery (FND-02):** All design tokens are declared ONCE as CSS custom properties on `:root` (dark, default) and `.light` / `[data-theme="light"]` in `app/globals.css`, then surfaced to Tailwind via `tailwind.config.js` `theme.extend` so they are consumable as utilities (`bg-background`, `text-primary`, `bg-glass`, `shadow-glow`, etc.). Hard-coded hex values in components are a contract violation — components consume tokens only.

## Spacing Scale

8-point scale (Tailwind default `spacing` already matches; declared explicitly as the contract). All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline chip padding, hairline insets |
| sm | 8px | Compact element spacing, button inner padding (y), badge padding |
| md | 16px | Default element spacing, button inner padding (x), card inner padding (mobile) |
| lg | 24px | Card inner padding (desktop), intra-section gaps |
| xl | 32px | Layout gaps, gap between sibling cards |
| 2xl | 48px | Major section breaks (vertical rhythm between sections, mobile) |
| 3xl | 64px | Page-level spacing, section vertical padding (desktop) |

Container: max content width `1200px` (Tailwind `max-w-6xl` ≈ 72rem is acceptable as the wrapper); horizontal page gutter `16px` mobile / `24px` tablet / `32px`+ desktop.

**Exceptions:**
- Interactive touch/click targets have a **minimum hit area of 44×44px** (WCAG 2.5.5 / Apple HIG) even when the visual glyph is smaller — applies to dock icons, theme toggle, icon-only buttons. This is a hit-area minimum, not a new spacing step.
- `1px` hairline borders (glass edge strokes) are a border width, not a spacing token.

## Typography

Geist Sans for all roles below unless noted. Exactly 4 sizes, exactly 2 weights (regular 400 + semibold 600). A single 700 is permitted ONLY for the hero wordmark "SOURABH JHA" (Display role, Phase 2) and is the documented exception — no other weights enter the system.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 600 (semibold) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.3 |
| Display | 28px | 600 (semibold) | 1.2 |

**Scale notes:**
- These four are the responsive *base*. The hero wordmark and large section titles scale UP fluidly above Display via `clamp()` (e.g. Display `clamp(28px, 5vw, 56px)`); they reuse the Display role's weight/line-height — they do not introduce new type roles.
- **Letter-spacing:** body/heading default (0). Mono "OS" accents (stat numbers, Mission Log IDs, palette text, terminal text) use Geist Mono with slight tracking `0.02em` and uppercase where it reads as a readout/label. Section eyebrow labels (e.g. "TRANSMISSION CENTER") are Label role, uppercase, tracking `0.08em`.
- **Mono is reserved** (see Design System) — never the body typeface.
- Min font size anywhere: 12px (mono micro-labels only, e.g. "MISSION LOG 01"); 12px is below the 14px Label step and is permitted ONLY for mono OS micro-labels.

## Color

Dichromatic minimalism — two hues do the work (near-black + cool ice-blue). No third brand hue, no neon/cyberpunk saturation. Premium dark theme with glassmorphism. **Dark-first**, with a full light theme required by FND-03; both themes are declared below.

### Dark theme (default)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#020202` | Page background, the void behind everything |
| Secondary (30%) | `#0B0D0F` raised surface + glass fills `rgba(255,255,255,0.04–0.08)` over the void | Cards, dock, sidebar, nav, panels, command palette surface |
| Accent (10%) | `#B2D5E5` (primary, ice-blue) | See reserved-for list below |
| Destructive | `#E5746A` (muted coral — desaturated to respect dichromatic minimalism) | Destructive/error only |

Text on dark: primary text `#EDF2F4` (near-white), secondary/muted text `rgba(237,242,244,0.64)`, disabled `rgba(237,242,244,0.38)`. Hairline/border `rgba(178,213,229,0.12)` (accent-tinted glass edge).

### Light theme (FND-03)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4F7F9` | Page background |
| Secondary (30%) | `#FFFFFF` surface + glass fills `rgba(2,2,2,0.03–0.06)` | Cards, dock, nav, panels |
| Accent (10%) | `#3E7C93` (darkened ice-blue so it passes AA on light surfaces — `#B2D5E5` is too pale on white for text/icons) | Same reserved-for list |
| Destructive | `#C0463B` | Destructive/error only |

Text on light: primary `#0B0D0F`, secondary `rgba(11,13,15,0.64)`, disabled `rgba(11,13,15,0.38)`. Hairline `rgba(2,2,2,0.10)`.

**Accent reserved for** (the 10% — never "all interactive elements"):
- Primary CTA buttons (`INITIATE TRANSMISSION`, primary actions)
- The active/selected state in the dock and command palette
- Focus rings (2px accent ring, keyboard focus — applies to both themes)
- Key data emphasis: stat-card numbers, the live status dot, animated card borders / hover-glow
- Links and the glow/aurora/mouse-follow lighting hue
- Brand marks (logo glyph, the "Career OS" wordmark accent)

Accent is NOT used for: body text, default card backgrounds, default borders, default icons (icons are text-color until active/hovered), or large fills. Saturated accent stays under ~10% of any viewport.

### Glass / Gradient / Glow tokens (FND-02 — required, consumable via Tailwind)

| Token | Dark value | Light value | Usage |
|-------|-----------|-------------|-------|
| `--glass-bg` | `rgba(255,255,255,0.05)` | `rgba(255,255,255,0.70)` | Glass card / dock / panel fill |
| `--glass-border` | `rgba(178,213,229,0.12)` | `rgba(2,2,2,0.08)` | Glass edge hairline (1px) |
| `--glass-blur` | `16px` | `16px` | `backdrop-filter: blur()` (use `backdrop-blur-md`; provide an opaque fallback layer for the `90+` Lighthouse / unsupported-backdrop case) |
| `--gradient-aurora` | `radial-gradient(at 30% 20%, rgba(178,213,229,0.18), transparent 60%)` | `radial-gradient(at 30% 20%, rgba(62,124,147,0.12), transparent 60%)` | Hero/background aurora & animated gradients |
| `--gradient-surface` | `linear-gradient(180deg, rgba(255,255,255,0.04), rgba(255,255,255,0))` | `linear-gradient(180deg, rgba(255,255,255,0.9), rgba(255,255,255,0.6))` | Subtle top-light on glass surfaces / reflections |
| `--glow-accent` | `0 0 24px rgba(178,213,229,0.35)` | `0 0 24px rgba(62,124,147,0.25)` | Hover glow, focus emphasis, active dock item (Tailwind `shadow-glow`) |
| `--glow-soft` | `0 8px 40px rgba(0,0,0,0.45)` | `0 8px 40px rgba(2,2,2,0.10)` | Ambient elevation under glass cards/dock |

**Contrast contract:** all text/icon pairs must meet WCAG AA (4.5:1 body, 3:1 large/UI) in BOTH themes — this is why light-theme accent darkens to `#3E7C93`. Verified at Phase 7's Lighthouse/a11y gate but enforced from Phase 1.

## Copywriting Contract

Voice: terse, confident, system/OS-flavored ("Career OS" persona) — short imperative verbs, no marketing fluff, ~80% visual / 20% text. Sentence case for prose; UPPERCASE reserved for OS labels/readouts (mono). Phase 1 ships only a placeholder section + theme toggle + global chrome, so most copy below is the inherited contract that later phases consume; the Phase 1 column flags what is in-scope NOW vs inherited.

| Element | Copy |
|---------|------|
| Primary CTA | Phase-specific, always verb + noun: `INITIATE TRANSMISSION` (contact, P6), `Download Resume` (palette/hero), `View Projects`. Phase 1 has NO marketing CTA — its only interactive control is the theme toggle. |
| Empty state heading | `Nothing here yet` |
| Empty state body | `This section is awaiting data. Content loads from the in-repo data files.` — used by any section whose data file holds only TODO placeholders (FND-07 seeds these). |
| Error state | `Something glitched in the system.` + recovery path: `Reload the page — if it persists, the console has details.` (form-level error pattern reused by Transmission Center validation in P6: inline field message `{Field} is required` / `Enter a valid email`.) |
| Destructive confirmation | None in Phase 1 — and effectively none site-wide. This is a read-only public portfolio; there are no delete/destroy actions. The simulated contact form does not persist or send data (TXC-02), so no destructive confirmation pattern is required. Destructive color tokens exist only for inline validation-error text. |

**Foundation-specific copy in scope NOW (Phase 1):**
- Theme toggle: icon-only (Lucide `Sun` / `Moon`), `aria-label="Toggle theme"`, no visible text label.
- Placeholder demo section heading (proves the motion wrapper + tokens): `System initialized` with body `Career OS foundation is online. Tokens, theme, and motion are live.`
- Document/meta: `<title>` `Sourabh Jha — AI Engineer / Backend Engineer`; brand wordmark `Career OS`.

## Motion Baseline (MOT-01 — foundation, inherited by all phases)

Framer Motion is the locked engine. Phase 1 ships ONE reusable section-reveal wrapper that every later section uses; it must respect `prefers-reduced-motion`.

| Aspect | Contract |
|--------|----------|
| Section reveal | Fade + 16px upward translate, `opacity 0→1`, `y: 16→0`. Duration `0.5s`, ease `[0.22, 1, 0.36, 1]` (easeOutExpo-like). Trigger on in-view (`whileInView`, `viewport={{ once: true, amount: 0.2 }}`). |
| Stagger | Children stagger `0.06s` for grouped reveals (cards, list items). |
| Hover/interaction | Scale `1.02` max + `--glow-accent`; transition `0.2s` ease-out. Subtle — never bouncy/playful. |
| Standard durations | micro `0.15s`, default `0.3s`, reveal `0.5s`, ambient/aurora loops `8–20s`. |
| Reduced motion (FND-05) | When `prefers-reduced-motion: reduce`, the wrapper renders content statically (no translate, instant or ≤0.01s opacity), all looping/parallax/mouse-follow effects are disabled. Implement via a `useReducedMotion()` gate in the shared wrapper so every consumer inherits it. |
| Performance (FND-06) | Animate only `transform` + `opacity` (GPU-friendly). No animating `width`/`height`/`top`/`left`/`box-shadow` in hot loops. `will-change` used sparingly. Target 90+ Lighthouse. |

## Responsive Baseline (FND-04)

| Breakpoint | Min width | Layout intent |
|-----------|-----------|---------------|
| mobile | 0 (base) | Single column, 16px gutters, dock collapses/compacts, stacked sections |
| tablet (`md`) | 768px | 24px gutters, 2-up card grids, dock horizontal |
| desktop (`lg`) | 1024px | 32px gutters, full multi-column layouts, 1200px max content width |
| wide (`xl`) | 1280px | Centered 1200px container with growing side whitespace |

Mobile-first Tailwind (`md:` / `lg:` / `xl:` prefixes). Layout reflows cleanly with no horizontal scroll at any width ≥320px. Tokens (color/spacing/type/glass) are identical across breakpoints — only layout/grid changes.
