## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm / shadcn registry -> local build | Third-party package code is fetched and executed at install/build time. This is the only meaningful attack surface in Phase 1. |
| repo -> public static site | Output is a static, public, frontend-only site. No auth, no backend, no server-side user input (the contact form is a later, simulated phase). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-SC | Tampering | npm + shadcn dependency installs | mitigate | Install only the audited set (next, react, react-dom, tailwindcss, framer-motion, lucide-react, next-themes, clsx, tailwind-merge) from the legitimacy table; commit the lockfile; shadcn from the official registry only. |
| T-01-01 | Information disclosure | repo / committed config | mitigate | No secrets, API keys, or tokens in code or env files this phase (static frontend); identity links are public by design. CI/local must not introduce `.env` secrets. |
| T-01-02 | Tampering | truncated next-swc binary (env gotcha) | accept | Known Windows/Node-box download truncation; remedy is documented (cache clean + reinstall). Not a security flaw, integrity restored by reinstall. |

No `@vercel/og` / dynamic image routes (avoids the known Node-box crash and reduces server-side surface). No `dangerouslySetInnerHTML`, no user input rendered this phase.
